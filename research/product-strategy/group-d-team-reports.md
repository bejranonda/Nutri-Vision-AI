**Decision 1: Launch Timing**

- **Risk:** Need to ensure 90%+ Thai food accuracy before launch

- **Recommendation:** Gemini API Year 1, evaluate custom model Year 2
- **Rationale:** Fast time-to-market, high accuracy, manageable costs ($0.002-0.005/image)
- **Risk:** Per-request costs scale with usage, but custom model requires $500K+ investment

**Decision 5: Geographic Expansion Timing**
- **Recommendation:** Thailand-only Year 1, Malaysia/Singapore Year 2
- **Rationale:** Focus beats fragmentation, establish brand before expanding
- **Risk:** HealthifyMe or others could enter Thailand first (unlikely given India focus)

## 3. FOR DEVELOPMENT TEAM

### Technical Priorities & Architecture Decisions

**Infrastructure Scalability:**

**Current:** Docker Compose (single server)
**Month 3:** Kubernetes (auto-scaling, handle 10K MAU)
**Month 6:** Multi-region deployment (Thailand + SEA ready)

**Database Optimization:**
- **Month 1:** PostgreSQL query optimization, proper indexing
- **Month 2:** Redis caching for food recognition results (reduce Gemini API calls)
- **Month 3:** Read replicas for analytics queries
- **Month 6:** Partitioning for large tables (food_scans, chat_messages)

**Google Gemini API Optimization:**
- **Cost Target:** <THB 5 per scan (currently ~THB 10-15)
- **Strategies:**
  1. Caching: Same food photo = cached result (Redis, 24h TTL)
  2. Image compression: 1MB ‚Üí 200KB before sending to API
  3. Batch requests: Combine multiple items in one API call
  4. Rate limiting: Prevent abuse (10 scans/hour for free tier)

**Performance Targets:**
- Scan-to-result: <5 seconds (target: 3 seconds)
- Page load: <2 seconds
- Time to Interactive (TTI): <3 seconds
- API response time: <500ms (95th percentile)
