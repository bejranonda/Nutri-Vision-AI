## 1. RICE Scoring Model

### 1.1 RICE Framework Explanation

**RICE** = (Reach Ã— Impact Ã— Confidence) / Effort

- **Reach:** How many users will this feature affect per time period? (e.g., per month)
- **Impact:** How much will this feature impact each user? (Scale: 0.25 = minimal, 0.5 = low, 1.0 = medium, 2.0 = high, 3.0 = massive)
- **Confidence:** How confident are we in our estimates? (Scale: 50% = low, 80% = medium, 100% = high)
- **Effort:** How many person-months will this take? (Engineering + design + testing)

## 3. Kano Model (Customer Satisfaction Analysis)

### 3.1 Kano Model Framework

The Kano Model categorizes features based on how they affect customer satisfaction:

- **Basic Needs (Threshold):** Must-haves. Customers expect these. Absence causes dissatisfaction.
- **Performance Needs (Linear):** More is better. Satisfaction increases with quality/quantity.
- **Excitement/Delight Needs:** Unexpected features that create joy. Absence doesn't cause dissatisfaction, but presence creates delight.
- **Indifferent:** Customers don't care either way.
- **Reverse:** Some customers prefer absence (e.g., ads, complex features).
